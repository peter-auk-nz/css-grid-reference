Opacity

Control transparency of elements and create layered visual effects

What is Opacity?

Opacity controls how transparent or opaque an element appears. It affects the entire element, including all its children and content. Values range from 0 (fully transparent) to 1 (fully opaque). Opacity is perfect for overlays, hover effects, and creating depth in designs.

Key Concept: Opacity affects the whole element, not just the background.

opacity Property

Sets the transparency level of an element.

.element { opacity: 0.5; }

Values: 0 to 1 (0 = invisible, 1 = fully visible)

Example: opacity: 0.5 makes element 50% transparent

Affects: Element and all descendants

Common Opacity Values

Standard opacity settings for different effects.

opacity: 1; /* Fully opaque (default) */
opacity: 0.8; /* Slightly transparent */
opacity: 0.5; /* Half transparent */
opacity: 0; /* Fully transparent (invisible) */

Tip: Use 0.7-0.9 for subtle transparency, 0.3-0.5 for overlays

Opacity vs RGBA

Two ways to create transparency with different behaviors.

/* Opacity - affects entire element */
.box { opacity: 0.5; }

/* RGBA - affects only background */
.box { background: rgba(255, 0, 0, 0.5); }

Opacity: Makes element AND children transparent

RGBA: Only background transparent, text stays opaque

Use RGBA when: You want transparent background with solid text

Hover Effects

Common use case for opacity in interactive elements.

.image { opacity: 1; transition: opacity 0.3s; }
.image:hover { opacity: 0.7; }

Pattern: Full opacity normally, reduce on hover for feedback

Transition: Smooth animation between states

Overlays

Create overlay effects for images and backgrounds.

.overlay { background: black; opacity: 0.5; }

Pattern: Dark or colored layer over content

Use: Improve text readability over images, create dramatic effects

Better approach: Use rgba() to avoid affecting child elements

Fade Animations

Animate opacity for fade in/out effects.

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

Use: Loading screens, modal appearances, content reveals

Disabled States

Visual indicator for disabled or inactive elements.

button:disabled { opacity: 0.4; cursor: not-allowed; }

Pattern: Reduce opacity to show element is not interactive

Common value: 0.4-0.6 for disabled states

Performance Considerations

Opacity is hardware-accelerated and performant.

/* Performant animation */
.element { transition: opacity 0.3s; }

Advantage: Animating opacity doesn't trigger layout recalculations

Combine with: transform for smooth, performant animations
